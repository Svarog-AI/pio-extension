# Task: Delete old skill and verify

Remove the `src/skills/test-driven-development/` directory and confirm no references remain. After deletion, only `tdd` remains as the TDD skill.

## Context

Steps 1 and 2 renamed all code references from `test-driven-development` to `tdd` and restructured prompts for iterative TDD. The old `src/skills/test-driven-development/` directory still exists with a single file (`SKILL.md`). This step completes the migration by removing it and verifying nothing breaks.

## What to Build

This is a deletion and verification step — no new code. Delete the old skill directory, confirm no stale references remain in `src/`, and verify type checking and tests pass.

### Approach and Decisions

- Use `rm -rf` or equivalent to delete `src/skills/test-driven-development/` (contains only `SKILL.md`).
- Run `grep -rn "test-driven-development" src/ --include="*.ts" --include="*.md"` to confirm zero results. The only match before deletion should be the old SKILL.md itself.
- Run `npm run check` (`tsc --noEmit`) to verify no type errors.
- Run `npx vitest run` to verify full test suite passes (750 tests as of Step 2).

The `.pio/PROJECT/` files (`OVERVIEW.md`, `ARCHITECTURE.md`, `CONVENTIONS.md`) still reference `test-driven-development`. These are auto-generated by the `project-context` capability — they will be regenerated on the next `/pio-project-context` run and are **out of scope** for this step. The grep acceptance criterion is scoped to `src/` only, matching the plan.
